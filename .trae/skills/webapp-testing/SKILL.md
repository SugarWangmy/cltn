# Webapp Testing

用于在修改 Web 页面/交互后，快速制定并执行高信心的测试与回归验证（适配原型页面、管理后台、复杂表格/弹窗/流程）。

## 何时调用

- 新增或修改交互流程（多步骤、弹窗、表单校验、状态机）
- 修复 bug 后需要证明"修好了且没回归"
- 用户说"帮我测一下/验证一下/回归一下"
- 需要把"验收标准"翻译成可执行的测试用例

## 输出目标

- **测试范围**：本次变更影响面与风险点
- **用例清单**：按优先级（P0/P1/P2）覆盖核心路径、边界、异常、空态
- **执行步骤**：清晰到点击/输入/预期结果
- **回归集合**：最小必要回归路径，避免漏测
- **证据**：关键截图点/日志点/可复现步骤

## 测试层级（先快后全）

### 1) 静态检查（快速）

- 构建/类型检查通过（若项目有）
- 控制台无明显报错（JS error / 资源 404）
- 关键交互入口可达（路由/hash/按钮）

### 2) 核心路径（P0）

覆盖用户最常用的主流程，必须全绿：

- 入口 → 关键操作 → 成功反馈 → 返回/关闭
- 关键数据刷新（例如欠款金额变化、列表排序变化）
- 阻断逻辑（例如缺字段/禁用项/欠款拦截）是否正确提示并阻止继续

### 3) 边界与异常（P1）

- 空输入、0、最大值、超上限、小数位、非法字符
- 网络/接口失败（若有）：错误提示、重试、不会写脏状态
- 空列表/无可选项：空态文案与按钮状态

### 4) 可用性与一致性（P2）

- 文案一致（按钮/标题/提示）
- 焦点与键盘：Esc 关闭弹窗、Enter 提交（若适用）
- 滚动：弹窗打开不抖动，关闭回到原位置（本仓库尤其重要）

## 表单/金额输入通用用例模板

- 输入：空 → 不报错；确认按钮禁用/提示（按设计）
- 输入：0 → 是否允许；显示格式是否正确
- 输入：最大值 → 允许且格式化
- 输入：超上限 → 自动 clamp 或提示错误（保持一致）
- 输入：非数字 → 自动清洗或提示错误
- 提交：成功后刷新展示值；取消不改变

## 列表/表格通用用例模板

- 排序：点一次降序，再点一次升序；排序标识一致
- 筛选：组合筛选/重置；空结果显示空态
- 粘性列/横向滚动：首列固定不抖动；表头与内容对齐

## 本仓库（ui-optimized.html）特化检查

- render() 全量重渲染后：事件绑定仍有效；lucide icons 正常渲染
- 滚动恢复：data-scroll-key 容器不回顶；window.scrollY 不抖动
- hash 路由：#PageId/StepId 切换无白屏；回退行为合理
- state 更新：确认/取消对 state 的影响正确；不会跨页面污染
