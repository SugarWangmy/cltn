# Update Docs

用于在代码变更后同步更新相关文档，确保文档与代码保持一致，提高项目的可维护性和可读性。

## 何时调用

- 代码变更后需要更新相关文档
- 新增功能或修改现有功能后需要更新README
- API接口变更后需要更新API文档
- 项目结构或配置变更后需要更新相关文档

## 输出目标

- **文档范围**：确定需要更新的文档文件
- **变更内容**：明确需要更新的具体内容
- **更新策略**：制定文档更新的步骤和方法
- **验证方法**：确保文档更新的准确性和完整性

## 文档类型

### 1) README文件

- 项目概述和功能介绍
- 安装和使用说明
- 配置选项和示例
- 贡献指南和许可证信息

### 2) API文档

- 接口定义和参数说明
- 返回值和错误处理
- 使用示例和最佳实践
- 版本变更记录

### 3) 技术文档

- 架构设计和流程图
- 组件说明和使用方法
- 部署和维护指南
- 故障排查和常见问题

## 更新流程

1. **识别变更**：分析代码变更的内容和影响范围
2. **确定文档**：确定需要更新的文档文件
3. **更新内容**：根据代码变更更新文档内容
4. **验证更新**：确保文档更新的准确性和完整性
5. **提交变更**：将文档更新与代码变更一起提交

## 最佳实践

- 文档更新与代码变更同步进行
- 使用清晰、简洁的语言描述功能和API
- 提供足够的示例和使用场景
- 定期检查文档的准确性和时效性
- 鼓励团队成员参与文档维护

## 推荐输出格式

### 需要更新的文档

- 文档名称：…
  - 更新内容：…
  - 变更原因：…

### 更新后的文档内容

- 关键部分的更新内容
- 新增或修改的示例
- 版本变更记录
