# 结算单模块需求文档

## 1. 产品概述

### 1.1 产品背景

结算单模块是美容门店与供应商之间财务结算的核心系统，用于管理赊销产品的销售结算、账期管理、支付处理等业务流程。该模块支持PC端和移动端操作，满足门店和总部不同的管理需求。

### 1.2 产品目标

- 实现赊销产品结算的规范化管理
- 支持多种结算方式（按销售、按周期、按账期）
- 提供清晰的账目明细和报表分析
- 实现支付流程的线上化处理
- 支持逾期提醒和自动对账

### 1.3 适用范围

- 适用对象：门店员工、店长、财务人员、供应商
- 使用设备：PC端、移动端
- 使用场景：日常结算对账、账期管理、支付处理、财务报表

---

## 2. 用户角色

| 角色 | 权限说明 |
|------|----------|
| 门店员工 | 查看结算单、发起支付申请 |
| 店长 | 审核结算单、审批支付 |
| 财务人员 | 核销结算单、处理支付、管理账期 |
| 供应商 | 查看结算明细、确认收款 |
| 管理员 | 系统配置、查看所有数据、报表管理 |

---

## 3. 功能需求

### 3.1 结算单管理

#### 3.1.1 结算单列表

- 显示所有结算单记录
- 支持多维度筛选：状态、时间、供应商、门店
- 状态筛选：待支付、已支付、逾期未支付、已核销
- 显示结算单基本信息：单号、供应商、金额、状态、时间

#### 3.1.2 结算单详情

- 显示结算单完整信息
- 商品明细列表：商品名称、数量、单价、金额
- 费用明细：货款、运费、其他费用
- 支付记录：支付时间、支付方式、支付金额
- 备注信息

#### 3.1.3 结算单创建

- 选择供应商
- 选择关联的销售订单
- 自动计算结算金额
- 设置结算周期和账期
- 添加备注说明

### 3.2 结算类型

#### 3.2.1 寄售结算

- 根据实际销售数量结算
- 按周期（月/周）生成结算单
- 支持库存扣减后自动结算

#### 3.2.2 赊销结算

- 基于赊销合同约定的结算
- 支持按账期结算
- 支持逾期利息计算

#### 3.2.3 采购结算

- 常规采购的结算管理
- 支持分批结算
- 支持预付款抵扣

### 3.3 支付管理

#### 3.3.1 支付申请

- 发起支付申请
- 选择支付方式：对公转账、微信、支付宝
- 上传支付凭证
- 提交审核

#### 3.3.2 支付审核

- 店长审批支付申请
- 财务复核支付凭证
- 支持驳回和重新提交

#### 3.3.3 支付记录

- 记录所有支付操作
- 显示支付状态和结果
- 支持支付凭证查看和下载

### 3.4 账期管理

#### 3.4.1 账期设置

- 为供应商设置账期（7天/15天/30天/60天/90天）
- 支持按商品类别设置不同账期
- 支持合同约定的特殊账期

#### 3.4.2 逾期管理

- 自动检测逾期未付账单
- 发送逾期提醒通知
- 记录逾期情况
- 逾期利息计算（可选）

### 3.5 报表分析

#### 3.5.1 结算汇总报表

- 按供应商汇总结算金额
- 按时间段汇总结算趋势
- 按门店汇总结算分布

#### 3.5.2 支付进度报表

- 显示各供应商支付进度
- 逾期账款统计
- 预计付款计划

## 4. 业务流程

### 4.1 标准结算流程

```
生成结算单 → 门店确认 → 财务审核 → 发起支付 → 支付审核 → 完成支付 → 供应商确认 → 核销归档
```

### 4.2 流程节点说明

| 节点 | 操作说明 | 参与角色 |
|------|----------|----------|
| 生成结算单 | 根据销售数据自动生成或手动创建 | 系统/财务 |
| 门店确认 | 核对结算明细，确认无误 | 门店员工 |
| 财务审核 | 审核结算金额和明细 | 财务人员 |
| 发起支付 | 提交支付申请 | 门店员工 |
| 支付审核 | 审批支付申请 | 店长/财务 |
| 完成支付 | 执行支付操作 | 财务人员 |
| 供应商确认 | 确认收到款项 | 供应商 |
| 核销归档 | 完成结算，存档记录 | 财务人员 |

### 5.2 结算单详情页

- 费用明细
- 支付记录
- 操作按钮：确认、支付、打印

### 5.3 支付弹窗

- 支付金额显示
- 支付方式选择
- 凭证上传
- 确认和取消按钮

## 6. 数据模型

### 6.1 核心数据

| 数据项 | 说明 |
|--------|------|
| 结算单号 | 系统生成唯一标识 |
| 供应商信息 | 供应商名称、联系人、联系方式 |
| 门店信息 | 门店名称、门店编号 |
| 结算金额 | 货款+费用-优惠 |
| 结算周期 | 月结/周结/即时 |
| 账期 | 应付日期 |
| 状态 | 待确认/待支付/已支付/已逾期/已核销 |
| 创建时间 | 结算单创建时间 |
| 支付时间 | 完成支付的时间 |

### 6.2 状态定义

| 状态 | 说明 |
|------|------|
| 待确认 | 结算单已生成，等待门店确认 |
| 待支付 | 已确认，等待支付 |
| 已支付 | 已完成支付，等待核销 |
| 逾期未支付 | 已过应付日期仍未支付 |
| 已核销 | 供应商确认，完成结算 |

---

## 7. 非功能性需求

### 7.1 性能要求

- 页面加载时间 ≤ 2秒
- 数据查询响应时间 ≤ 1秒
- 支持大数据量展示（1000+条记录）

### 7.2 兼容性要求

- 支持Chrome、Firefox、Safari、Edge最新版本
- 适配1920×1080及以上分辨率
- 移动端适配iOS和Android

### 7.3 安全要求

- 用户认证和权限控制
- 操作日志完整记录
- 敏感数据加密存储
- 支付凭证安全存储
