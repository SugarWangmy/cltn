# 赊销铺货门店管理模块需求文档

## 1. 产品概述

### 1.1 产品背景

赊销铺货是美容行业供应链管理的重要模式，指供应商先将商品铺货到门店销售，待商品销售后再与供应商结算货款。这种模式可以降低门店资金压力，帮助供应商拓展市场渠道。本模块用于管理赊销铺货业务中的门店信息、合同管理、铺货额度、结算对账等核心业务流程。

### 1.2 产品目标

- 实现赊销铺货门店的规范化管理
- 支持铺货额度控制和信用评估
- 提供完整的合同生命周期管理
- 实现销售数据的实时同步和对账
- 支持多供应商、多门店的复杂业务场景

### 1.3 适用范围

- 适用对象：供应商业务员、门店店主、总部管理人员、财务人员
- 使用设备：PC端、移动端
- 使用场景：门店拓展、合同签订、铺货管理、结算对账、账期管理

## 2. 用户角色

| 角色 | 权限说明 |
|------|----------|
| 供应商业务员 | 负责开拓门店、签订合同、跟进销售数据 |
| 门店店主 | 查看铺货商品、管理销售、发起结算 |
| 区域经理 | 管理区域内门店、审核合同、监控业绩 |
| 财务人员 | 审核结算单、处理付款、管理账期 |
| 管理员 | 系统配置、权限管理、全局数据查看 |

## 3. 功能需求

### 3.1 门店管理

#### 3.1.1 门店信息维护

- 门店基础信息：名称、地址、联系人、联系电话
- 门店资质：营业执照、经营许可证
- 门店规模：营业面积、床位数量、员工人数
- 营业信息：营业时间、经营范围

#### 3.1.2 门店状态管理

- 门店状态：待审核、正常合作、暂停合作、终止合作
- 状态变更记录
- 暂停/终止合作原因记录

#### 3.1.3 门店查询筛选

- 支持按区域、状态、等级筛选
- 支持关键字搜索（门店名称、联系人）
- 支持地图模式查看门店分布

### 3.2 合同管理

#### 3.2.1 合同签订

- 选择签约门店
- 选择关联供应商
- 设置合同条款：
  - 铺货额度（信用额度）
  - 账期（7天/15天/30天/60天/90天）
  - 结算方式（月结/季结/即时结）
  - 返点政策
  - 违约条款
- 合同附件上传（营业执照、资质证明）
- 合同审批流程

#### 3.2.2 合同变更

- 合同内容变更申请
- 额度调整审批
- 账期调整审批
- 变更记录存档

#### 3.2.3 合同终止

- 合同到期提醒
- 终止申请和审批
- 清算对账
- 终止存档

### 3.3 铺货管理

#### 3.3.1 铺货额度

- 信用额度设置：根据门店资质和销售能力评估
- 额度使用情况实时展示
- 额度预警提醒
- 临时额度申请和审批

#### 3.3.2 铺货商品

- 铺货商品目录管理
- 商品分类：一级分类、二级分类
- 商品信息：名称、规格、单位、供货价、零售价
- 库存展示：当前库存、销售中、已结算

#### 3.3.3 铺货发货

- 铺货申请创建
- 选择商品和数量
- 仓库发货确认
- 门店收货确认
- 铺货记录查询

### 3.4 销售管理

#### 3.4.1 销售数据上报

- 门店销售数据录入
- 销售数据核对
- 销售趋势分析
- 热销商品排行

#### 3.4.2 库存管理

- 实时库存查询
- 库存预警设置
- 临期商品提醒
- 库存盘点支持

#### 3.4.3 销售结算

- 按销售结算：卖多少结多少
- 按周期结算：按约定周期结算
- 按账期结算：销售后若干天结算

### 3.5 账期与结算

#### 3.5.1 账期管理

- 账期设置：按合同约定
- 账期提醒：提前3天/1天提醒
- 逾期处理：逾期利息、暂停供货
- 账期延长申请

#### 3.5.2 结算对账

- 生成对账单
- 核对销售数据
- 核对退货数据
- 确认结算金额
- 确认后生成结算单

#### 3.5.3 支付管理

- 支付申请
- 支付审核
- 支付记录
- 支付凭证管理

### 3.6 数据分析

#### 3.6.1 销售分析

- 销售额趋势
- 销售占比分析
- 门店业绩排行
- 商品销售排行

#### 3.6.2 应收账款

- 应收账款统计
- 账龄分析
- 逾期账款提醒
- 坏账统计

#### 3.6.3 业绩报表

- 门店业绩报表
- 供应商业绩报表
- 区域业绩报表
- 自定义报表

## 4. 业务流程

### 4.1 门店入驻流程

```
门店申请 → 资质审核 → 信用评估 → 合同签订 → 额度设置 → 开通铺货 → 正式合作
```

### 4.2 铺货销售流程

```
门店申请铺货 → 总部审批 → 仓库发货 → 门店收货 → 销售上报 → 数据核对 → 发起结算 → 支付完成
```

### 4.3 结算对账流程

```
生成对账单 → 门店确认 → 财务审核 → 确认结算金额 → 生成结算单 → 发起付款 → 付款完成 → 核销归档
```

### 4.4 流程节点说明

| 节点 | 操作说明 | 参与角色 |
|------|----------|----------|
| 门店申请 | 门店提交入驻申请和资质材料 | 门店店主 |
| 资质审核 | 审核门店经营资质 | 区域经理 |
| 信用评估 | 评估门店信用等级和额度 | 风控/总部 |
| 合同签订 | 签订赊销铺货合同 | 业务员/店主 |
| 额度设置 | 设置铺货信用额度 | 总部 |
| 铺货发货 | 仓库根据订单发货 | 仓库 |
| 门店收货 | 门店确认收到货物 | 门店 |
| 销售上报 | 门店上报销售数据 | 门店 |
| 结算对账 | 核对销售数据 | 门店/财务 |
| 支付完成 | 完成货款支付 | 财务 |

## 6. 数据模型

### 6.1 核心数据

| 数据项 | 说明 |
|--------|------|
| 门店ID | 系统唯一标识 |
| 门店名称 | 门店官方名称 |
| 联系人 | 门店负责人 |
| 联系电话 | 联系方式 |
| 经营地址 | 门店地址 |
| 信用等级 | A/B/C/D级 |
| 铺货额度 | 最大铺货金额 |
| 已用额度 | 当前已使用额度 |
| 账期 | 应付账期限 |
| 合同状态 | 正常/暂停/终止 |
| 合作开始日期 | 首次合作时间 |
| 合同到期日期 | 合同结束时间 |

### 6.2 状态定义

| 状态 | 说明 |
|------|------|
| 待审核 | 提交入驻申请，等待审核 |
| 正常合作 | 已签订合同，正常经营 |
| 暂停合作 | 暂时停止供货，等待处理 |
| 终止合作 | 结束合作关系 |

### 6.3 合同状态

| 状态 | 说明 |
|------|------|
| 草稿 | 合同拟制中 |
| 待审核 | 等待审批 |
| 生效中 | 合同已生效执行 |
| 已变更 | 合同内容已变更 |
| 已到期 | 合同已到期 |
| 已终止 | 合同已终止 |

## 7. 非功能性需求

### 7.1 性能要求

- 页面加载时间 ≤ 2秒
- 数据查询响应时间 ≤ 1秒
- 支持10000+门店数据管理
- 支持100+供应商同时在线

### 7.2 兼容性要求

- 支持Chrome、Firefox、Safari、Edge最新版本
- 适配1920×1080及以上分辨率
- 移动端适配iOS和Android

### 7.3 安全要求

- 用户身份认证和权限控制
- 操作日志完整记录
- 合同文件安全存储
- 敏感数据加密

### 7.4 可用性要求

- 系统可用性 ≥ 99.9%
- 数据定期备份
- 故障恢复机制
