# 管理后台模块需求文档

## 1. 产品概述

### 1.1 产品背景

管理后台是整个美容门店业务系统的管理中心，为企业管理者提供全面的运营管理、数据统计、系统配置等功能。该系统基于PC端操作，支持多角色权限管理，满足日常运营管理的各项需求。

### 1.2 产品目标

- 实现门店运营数据的集中管理和可视化展示
- 提供完整的业务流程管理工具
- 支持多维度数据分析和报表输出
- 实现精细化的权限管理和操作审计
- 提高管理效率，降低运营成本

### 1.3 适用范围

- 适用对象：企业管理人员、运营人员、财务人员、IT运维人员
- 使用设备：PC端（桌面浏览器）
- 使用场景：日常运营管理、数据统计分析、系统配置维护

---

## 2. 用户角色

| 角色 | 权限说明 |
|------|----------|
| 超级管理员 | 系统最高权限，可管理所有功能和数据 |
| 运营管理员 | 运营数据管理、活动管理、客户管理 |
| 财务管理员 | 财务管理、账务核对、报表查看 |
| 门店店长 | 本门店数据查看、员工管理、简单操作 |
| 客服人员 | 客户信息管理、预约管理、咨询处理 |

## 3. 功能需求

### 3.1 首页仪表盘

#### 3.1.1 经营概览

- 今日营业数据：销售额、订单数、客单价
- 会员数据：新会员数、活跃会员数
- 库存数据：库存预警数量、待收货数量
- 待处理事项：待审核订单、待处理售后

#### 3.1.2 数据统计

- 销售趋势图（按日/周/月）
- 品类销售占比
- 门店业绩排行
- 畅销商品排行

#### 3.1.3 快捷入口

- 待办事项列表
- 常用功能快捷入口
- 通知公告

### 3.2 订单管理

#### 3.2.1 订单列表

- 显示所有订单记录
- 支持多维度筛选：订单状态、时间范围、支付方式、门店
- 订单状态：待支付、已支付、已完成、已取消、售后中
- 支持订单搜索（订单号、客户姓名、手机号）

#### 3.2.2 订单详情

- 订单基本信息：订单号、下单时间、支付方式
- 商品信息：商品名称、数量、单价
- 客户信息：姓名、电话、会员等级
- 操作记录：订单状态变更历史

#### 3.2.3 订单操作

- 订单核销：验证核销码完成服务
- 订单退款：处理退款申请
- 订单修改：修改订单信息（限特定状态）
- 订单导出：支持Excel导出

### 3.3 客户管理

#### 3.3.1 客户列表

- 显示所有客户信息
- 支持搜索和筛选：姓名、电话、会员等级、注册时间
- 客户基本信息：姓名、电话、性别、生日、会员等级

#### 3.3.2 客户详情

- 客户档案：基本信息、会员卡信息、消费记录
- 服务记录：历史服务项目、时间、技师
- 消费分析：消费金额、消费频次、偏好项目

#### 3.3.3 会员管理

- 会员等级设置：等级名称、升级条件、权益配置
- 会员积分规则：积分获取、积分抵扣
- 会员营销活动：优惠券、满减活动、会员日

### 3.4 库存管理

#### 3.4.1 商品管理

- 商品列表：显示所有商品信息
- 商品分类：一级分类、二级分类
- 商品信息：名称、条码、规格、单价、库存

#### 3.4.2 库存管理

- 库存查询：按商品、仓库、门店查询
- 库存预警：设置库存下限，自动提醒
- 库存调拨：门店间商品调拨

#### 3.4.3 盘点管理

- 盘点计划：创建盘点任务
- 盘点执行：扫码盘点、数据录入
- 盘点报告：盈亏统计、差异分析

### 3.5 财务管理

#### 3.5.1 收款管理

- 收入记录：所有收入明细
- 收入统计：按日/周/月汇总
- 对账管理：日对账、月对账

#### 3.5.2 付款管理

- 供应商付款记录
- 付款申请和审批
- 付款凭证管理

#### 3.5.3 财务报表

- 利润表
- 资产负债表
- 现金流量表
- 自定义报表

### 3.6 门店管理

#### 3.6.1 门店信息

- 门店列表：显示所有门店
- 门店信息：名称、地址、联系电话、营业时间
- 门店配置：服务项目、员工排班

#### 3.6.2 员工管理

- 员工列表：显示所有员工
- 员工信息：姓名、工号、岗位、联系方式
- 角色权限：岗位角色、功能权限

#### 3.6.3 考勤管理

- 考勤记录：签到签退记录
- 考勤统计：出勤率、迟到早退统计

### 3.7 系统管理

#### 3.7.1 权限管理

- 用户管理：创建用户、分配角色
- 角色管理：定义角色、配置权限
- 权限配置：功能权限、数据权限

#### 3.7.2 系统配置

- 基础参数：系统名称、 logo、主题色
- 支付配置：支付方式、费率
- 消息配置：短信、推送、邮件

#### 3.7.3 日志管理

- 操作日志：用户操作记录
- 登录日志：用户登录记录
- 系统日志：系统运行日志

## 4. 业务流程

### 4.1 订单处理流程

```
客户下单 → 订单支付 → 订单确认 → 服务执行 → 订单核销 → 完成评价 → 数据统计
```

### 4.2 退款处理流程

```
客户申请退款 → 客服审核 → 财务审核 → 执行退款 → 退款完成 → 记录归档
```

### 4.3 库存调拨流程

```
发起调拨 → 调出仓库确认 → 物流运输 → 调入仓库接收 → 完成调拨 → 更新库存
```

- 底部状态栏：版本信息、帮助链接

### 5.2 列表页设计

- 筛选区域：多条件筛选
- 表格区域：数据列表展示
- 分页控件：翻页、每页条数
- 操作区域：新增、导出、批量操作

### 5.3 详情页设计

- 基本信息卡片
- 详细信息标签页
- 操作记录时间线
- 相关联的其他信息

## 6. 非功能性需求

### 6.1 性能要求

- 页面首次加载时间 ≤ 3秒
- 数据查询响应时间 ≤ 2秒
- 支持10000+条数据列表展示
- 支持100+并发用户

### 6.2 兼容性要求

- 支持Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- 最佳分辨率：1920×1080
- 最小支持分辨率：1366×768

### 6.3 安全要求

- 用户身份认证（用户名密码、短信验证码）
- 角色权限控制（功能权限、数据权限）
- 操作日志完整记录
- 敏感数据加密存储和传输
- 密码强度要求和安全策略

### 6.4 可用性要求

- 系统可用性 ≥ 99.9%
- 数据备份策略（每日备份、异地备份）
- 故障恢复时间 ≤ 30分钟
