# 商城收货模块需求文档

## 1. 产品概述

### 1.1 产品背景

商城收货是美容门店供应链管理的重要环节，用于确认供应商配送商品的接收情况。本模块支持移动端操作，员工可通过扫描商品条码确认收货，确保商品入库的准确性和及时性。

### 1.2 产品目标

- 实现收货流程的数字化管理，提高工作效率
- 支持扫码快速识别商品，减少手动输入错误
- 实时记录收货数据，确保库存准确
- 支持批量收货，提升操作效率

### 1.3 适用范围

- 适用对象：美容门店员工、仓库管理人员
- 使用设备：移动端设备（手机、平板）
- 使用场景：供应商配送收货、总部调拨收货、采购退货收货

---

## 2. 用户角色

| 角色 | 权限说明 |
|------|----------|
| 普通员工 | 执行收货操作、扫描商品、确认收货 |
| 仓库管理员 | 审核收货数据、处理异常收货 |
| 店长 | 审批异常收货、处理退货申请 |
| 管理员 | 系统配置、查看所有收货记录 |

---

## 3. 功能需求

### 3.1 收货任务管理

#### 3.1.1 任务列表

- 显示待收货任务列表
- 显示任务状态：待收货、收货中、已完成、异常
- 支持按时间、供应商、状态筛选
- 显示任务紧迫程度提示

#### 3.1.2 任务详情

- 显示供应商信息
- 显示商品清单列表
- 显示期望收货数量
- 显示配送单号信息

### 3.2 扫码收货

#### 3.2.1 商品扫描

- 支持扫描商品条码识别商品
- 支持扫描箱码批量识别
- 扫描后显示商品信息（名称、规格、期望数量）
- 支持手动输入商品编码作为备选

#### 3.2.2 数量确认

- 显示期望收货数量
- 支持输入实际收货数量
- 支持标记异常数量（损坏、少送、多送）
- 支持添加备注说明

### 3.3 收货确认

#### 3.3.1 收货检查

- 显示已收货商品列表
- 标记未收货商品
- 显示差异商品统计
- 支持再次扫描补扫

#### 3.3.2 提交确认

- 确认收货信息完整
- 提交收货数据
- 生成收货记录
- 触发库存更新

### 3.4 异常处理

#### 3.4.1 异常类型

- 数量异常：实际数量与期望不符
- 商品异常：商品损坏、规格错误
- 配送异常：配送单信息不符

#### 3.4.2 异常处理流程

- 记录异常情况
- 拍摄异常商品照片
- 提交异常处理申请
- 等待审批处理

---

## 4. 业务流程

### 4.1 标准收货流程

```
开始 → 查看收货任务 → 扫描商品 → 确认数量 → 提交收货 → 审核确认 → 更新库存 → 结束
```

### 4.2 异常收货流程

```
开始 → 扫码发现异常 → 标记异常商品 → 拍摄照片 → 提交异常申请 → 店长审批 → 处理结果 → 结束
```

## 5. 界面设计

### 5.1 收货首页

- 顶部：页面标题“商城收货”
- 待收货任务卡片列表
- 每个卡片显示：供应商名称、商品数量、期望时间

### 5.2 扫码页面

- 扫描区域：摄像头取景框
- 扫描提示文字
- 手动输入入口

### 5.3 收货确认页

- 商品列表：显示已扫描商品
- 数量编辑：每行可修改实际数量
- 底部操作：确认提交按钮

---

## 6. 数据需求

### 6.1 核心数据

| 数据项 | 说明 |
|--------|------|
| 收货单号 | 系统生成唯一标识 |
| 供应商信息 | 供应商名称、联系方式 |
| 商品信息 | 商品名称、条码、规格 |
| 期望数量 | 订单配送数量 |
| 实际数量 | 实际收货数量 |
| 收货时间 | 完成收货的时间 |
| 操作人 | 收货操作人员 |

### 6.2 状态定义

| 状态 | 说明 |
|------|------|
| 待收货 | 任务已创建，等待收货 |
| 收货中 | 正在进行收货操作 |
| 已完成 | 收货已完成，数据已提交 |
| 异常 | 存在异常待审批处理 |

## 7. 非功能性需求

### 7.1 性能要求

- 页面加载时间 ≤ 2秒
- 扫码响应时间 ≤ 500毫秒
- 支持离线缓存，网络恢复后自动同步

### 7.2 兼容性要求

- 支持iOS 12.0及以上
- 支持Android 8.0及以上
- 适配主流屏幕尺寸

### 7.3 安全要求

- 用户登录验证
- 操作日志记录
- 数据传输加密
